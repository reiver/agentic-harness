Agentic Harness

the secure runtime environment wrapper that makes use of LLM(s)

by Charles Iliya Krempeaux

An LLM can be used to create an AI agent — but, an LLM by itself is not an AI agent. An LLM by itself either behaves as a chaotic, autocomplete engine (if it is a raw LLM) or behaves as an AI assistant chatbot (if it has gone through post-training refinement).

To have an AI agent one needs an Agentic Harness (which wraps an LLM). Either an Agentic Harness that someone else made, or an Agentic Harness that you make yourself.

The Agentic Harness calls and makes use of the LLM. Sort of similar to how a computer makes use of a CPU. You can sort of think of the relationship between the Agentic Harness and the LLM this way —

The Agentic Harness is to the LLM, as the computer is to the CPU.

It isn't a perfect analogy, but it probably gives you a good enough intuition.

Agentic Harnesses often deal with:

  • context management
  • statement management
  • sandboxing (for security)
  • guardrailing
  • interceptors

Examples

One example of an Agentic Harness is Claude Code (and alternatives to it such as Aider, Crush, OpenCode, and others). Claude Code is an Agentic Harness that wraps the Claude Model(s).

Another example of an Agentic Harness are what you create using the Eino Agentic Framework. Where with Claude Code (and Aider, Crush, OpenCode, and others) you have to use those Agentic Harnesses as is — with Eino Agentic Framework, you can create custom Agentic Harnesses.

Agentic AI Engineering

If you are doing Agentic AI Engineering, then you are likely creating cuatom Agentic Harnesses — either from scratch, or using frameworks such as the Eino Agentic Framework, LangGraph, LangChain, and others.
